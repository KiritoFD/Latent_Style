## 2. 研究目标与核心矛盾

本项目围绕 VAE 潜空间上的图像风格迁移展开。目标不是只提高单一风格分数，而是在以下几项之间寻求稳定平衡：

- 风格一致性
- 内容结构保持
- 颜色与亮度稳定
- 输出图像的整体分布质量

随着实验推进，一个越来越清楚的事实是：这个任务的难点不是“模型能不能学到风格”，而是“如何在强化风格表达时，不牺牲内容结构、颜色约束和分布质量”。因此，项目后半程的重点逐渐从“有没有风格迁移能力”转向“如何构造更可控的风格注入和更合理的损失平衡”。

## 3. 模型演进主线

根据 git 历史与源码差异，项目可以分为四个主要阶段。

### 3.1 阶段 A：参考图条件化原型

代表提交：
- `ae596d1`

这一阶段的模型仍然依赖 `style_ref`。也就是说，推理时必须显式输入参考风格图，模型本质上仍是参考图条件化方案。此时已有 `AdaGN` 一类基础调制结构，但风格信息尚未内嵌为稳定的参数化表示。

这一阶段的重要意义不在于最终形态，而在于证明了两件事：
- 在潜空间做风格迁移是可行的。
- 风格调制类结构比纯粹把损失叠上去更值得作为主线。

### 3.2 阶段 B：风格 ID 与空间先验引入

代表提交：
- `d916277`
- `83ffe10`
- `1f818cc`
- `f7b328c`

这是第一个真正的结构转折点。核心变化是从“推理时携带参考图”转向“把风格蒸馏进模型”。在源码层面，`style_emb`、`style_spatial_id_16`、`style_spatial_id_32` 等组件进入主线，风格不再只是外部输入，而成为模型内部可学习的全局表示与空间先验。

这一阶段的设计逻辑可以概括为：
- 用 `style_emb` 承担全局风格身份。
- 用 `style_spatial_id_*` 承担更粗粒度的空间风格布局。
- 用多 patch SWD 等分布约束，防止模型只学到粗浅颜色偏置而失去笔触和局部纹理统计。

这一阶段的核心贡献，是把项目从“参考图驱动的原型”推到了“具备独立风格表示能力的模型”。

### 3.3 阶段 C：TextureDict、Skip 路由与 Decoder 重构

代表提交：
- `c619fda`
- `4992e06`
- `4699637`
- `ed596c0`

随着风格表示能力逐渐稳定，问题开始转移到“风格到底通过什么路径注入网络，哪些路径会导致 identity shortcut 泄漏，decoder 如何避免把源域细节直接拷回去”。这一阶段的设计重心因此转向：

- `TextureDictAdaGN`
- `NormFreeModulation`
- `StyleAdaptiveSkip` / `StyleRoutingSkip`
- no-norm decoder
- 更系统的 NCE / SWD / color loss 配方

这一步非常关键，因为它把“是否有风格注入”进一步拆成了三个层面：
- 调制器本身如何表达风格
- skip connection 是否会把源图高频直接偷渡回来
- decoder 是在“生成风格化结果”，还是在“恢复源图细节并顺带染色”

这也是项目开始形成真正实验体系的阶段：A 系列参数消融、注入路径消融、decoder 结构组、NCE/SWD 路线组，都在这一阶段逐步成型。

### 3.4 阶段 D：Cross-Attention 到 CGW Backbone

代表提交：
- `c8577e0`
- `426ae0a`
- `cfdbaba`
- `c405b9d`
- `4e166f0`

进入这一阶段后，项目从“局部调制结构优化”进一步转向“特征交互和 backbone 组织方式”的系统搜索。`CrossAttnAdaGN`、`SpatialSelfAttention`、`AttentionBlock`、`SemanticCrossAttn`、conv/global/window 混合 backbone、shifted window 修补、micro-batch 训练范式等先后进入主线。

这一阶段的特点是：
- 风格注入开始具有更明确的 token 级或空间交互特征。
- attention 不再只是可选增强块，而是进入 backbone 搜索空间。
- 训练组织本身成为性能关键点，micro-batch 的引入说明工程约束已经反过来影响模型上限。

因此，这一阶段不只是“换了几个模块”，而是把研究问题扩大到了“在显存与训练稳定性约束下，怎样组织一个更强的风格迁移骨架”。

## 4. 核心模块的演进逻辑

纵观整个历史，真正持续演进的核心不是某一个固定 backbone，而是风格调制链条。

### 4.1 风格调制器是主线中的主线

从 `AdaGN` 到 `TextureDictAdaGN` 再到 `CrossAttnAdaGN`，可以看到模型主线一直在回答同一个问题：风格应当如何被表示、路由并作用于特征。

这背后的认识逐渐清晰：
- 仅靠全局风格向量不够。
- 仅靠空间图先验也不够。
- 仅靠把注入位置挪来挪去也不够。
- 真正重要的是“表示 + 路由 + 约束”三者一起设计。

### 4.2 Skip 与 Decoder 决定是否会走捷径

skip 与 decoder 一开始更像附属结构，但到中后期已经变成关键研究对象。实验结果说明，很多 style 指标的提升来自削弱 identity、放大 skip、或修改 decoder 后形成的“捷径式风格化”，这种提升并不稳定，往往伴随 LPIPS、FID、art-FID 的退化。

因此，这条线的价值并不只是提升指标，而是帮助识别哪些结构会诱导模型偷内容或过度染色。

### 4.3 Backbone 搜索是后期的扩展方向

conv/global/window 混合、attention block 和 c-g-w backbone 的引入，代表项目后半程开始探索更强的特征建模能力。但从历史看，backbone 的重要性略低于调制器与约束系统本身。换句话说，backbone 是后期增益放大器，而不是最早建立任务可解性的核心原因。

## 5. Loss 系统的演进

Loss 的演进大致经历了三个层次。

### 5.1 从“先让任务成立”到“保护内容结构”

早期损失更多承担基础约束作用，确保模型不会完全失控。随着项目推进，NCE 被重新确认为有效，这说明内容结构保持不能只依赖重建类约束，还需要更具有判别性的局部/对应关系约束。

### 5.2 从“有风格”到“有分布感”

SWD 路线的逐渐稳定，说明项目不满足于简单风格信号，而开始希望模型在更细粒度的局部分布上接近目标域。多 patch、多尺度、domain vs instance 的比较，都是围绕“怎样让生成结果更像目标域分布”展开的。

### 5.3 从“控制风格”到“控制代价”

后期 color loss、亮度约束、identity 权重、TV 权重、学习率等联合优化越来越重要。这说明 Loss 系统已经不再是补充项，而是整个模型行为的控制面板。特别是高风格分数与内容/分布质量之间的张力，让 Loss 配比本身成为核心研究问题。

## 7. 代表性实验与阶段性认识

下面列出几组最具有解释力的结果，它们不一定是单项冠军，但最能说明项目的认识是怎样形成的。

### 7.1 `exp_S1_zero_id`：高 style 分数并不等于好模型

`exp_S1_zero_id` 的 style 分数很高，但 LPIPS、FID、art-FID 全面恶化。它是项目中的一个典型反例，说明一旦 identity 约束被拿掉，模型确实会更愿意冲高风格指标，但代价是内容结构和整体分布质量显著受损。

这个实验的重要性非常高，因为它帮助项目尽早摆脱了“只追 style 分数”的错误优化方向。

### 7.2 `exp_G1_edge_rush`：主线实验中非常强的均衡点

从 FID、art-FID 和综合均衡角度看，`exp_G1_edge_rush` 是一类非常值得重视的结果。它说明主线实验体系内部其实已经出现了较强的平衡候选，而不是只有后期复杂结构才可能得到有效结果。

这意味着项目的主干设计是成立的，后续很多工作更像是在扩大稳定优势，而不是从零开始找方向。

### 7.3 `exp_3_macro_strokes`：艺术域分布质量的重要证据

在 art-FID 维度上，`exp_3_macro_strokes` 的表现非常值得强调。它表明宏观笔触相关设计对“更像目标艺术域分布”是有帮助的。即使它不是 style 冠军，它仍然是很好的阶段成果，因为它对应的是更加研究导向的结论，而不是单纯追一个分数。

### 7.4 A 系列：项目从经验调参走向可解释调参

A 系列参数消融的价值在于，它们不是一次性的“调好了就算”，而是建立了 patch、identity、TV 等参数与结果行为之间的对应关系。对于答辩来说，这类实验非常重要，因为它说明当前配方并非拍脑袋，而是经过单因素验证收敛出来的。

### 7.5 `style_oa_*` 与曲线分析：最优 checkpoint 不是最后一个

对 `style_oa_5_lr5e4_wc2_swd60_id30_e120_interval10` 等实验的 `summary_history` 分析显示，很多实验在 30 epoch 或中期 checkpoint 已经达到更好的平衡点，而继续训练只会推高 style、恶化 LPIPS 或 FID。

这条结论说明：
- 项目的评价方式必须从“最后一个 checkpoint”转向“曲线视角”
- 后期训练过程存在明显的风格过拟合或结构漂移现象

这属于中期阶段非常有价值的研究认识。

## 8. 当前最稳定的阶段性结论

截至目前，可以较为稳定地提出以下结论。

### 8.1 风格调制是主导矛盾

真正决定模型上限的，不只是 backbone 深浅，而是风格如何表示、如何注入、如何受损失约束。

### 8.2 单项最高 style 往往不可靠

项目已经多次观察到，style 冲高往往伴随 identity 弱化、颜色退化、分布恶化。仅看 style 指标会误导模型选择。

### 8.3 实验系统已经足够支撑结构化结论

从主线实验、单因素消融、路径消融到联合优化，项目已经形成了完整实验链路。中期答辩不需要把自己表述成“还在零散试错”，而应该明确说明已经建立了结构化验证体系。

### 8.4 后期工作的重点是稳定与均衡，而不是继续无上限堆复杂度

历史材料显示，许多失败不是因为模型不够复杂，而是因为均衡点没有被稳定抓住。因此下一阶段更值得投入的是：
- checkpoint 选择机制
- 多指标联合排序
- 风格增强与内容保持的更稳健平衡
- attention backbone 与 loss 系统的协同，而不是单独加模块

## 9. 中期答辩可使用的叙事框架

如果把整个项目压缩成一条适合口头汇报的故事线，可以表述为：

1. 早期先证明潜空间风格迁移可行。
2. 然后摆脱参考图依赖，把风格蒸馏进模型，建立可学习的全局风格表示和空间先验。
3. 再进一步发现，问题不只是“有没有风格表示”，而是风格如何通过调制器、skip、decoder 真正进入网络且不走捷径。
4. 在此基础上，项目进入 attention backbone 与联合优化阶段，开始系统搜索更强架构和更稳损失配方。
5. 与此同时，实验体系逐渐从主线对照扩展到单因素消融、路径消融、曲线分析和多指标综合排序。
6. 当前最重要的阶段成果，是已经识别出风格增强与内容/分布退化之间的稳定张力，并据此建立起更成熟的实验评价框架。

## 10. 下一阶段工作建议

基于当前资料，可以把后续工作明确分成三条线。

### 10.1 模型线

### 10.3 文档线

## 11. 总结

从现有材料看，`Cycle-NCE` 已经不是一个靠零散试验推进的项目，而是一个围绕风格调制、结构约束、loss 配比和实验系统化验证逐步演进的研究过程。模型主线经历了参考图条件化、风格内嵌、空间先验、结构化调制、attention backbone 与 micro-batch 训练组织等阶段；实验主线则从基线对照逐步扩展到单因素消融、路径验证、曲线分析与多指标均衡评价。

因此，中期答辩阶段最值得强调的，不只是“目前最好结果是多少”，而是项目已经形成了清晰的研究对象、稳定的演进逻辑和足以支撑下一阶段深入优化的实验体系。
