# 实验曲线类型学

用途：把 `summary_history` 看成“训练动态证据”，而不只是若干 checkpoint。

## 1. 为什么做类型学

同一类模型失败方式经常重复出现。  
如果把这些曲线模式提前归类，后面复盘会快很多。

## 2. 类型 A：中期甜点型

定义：

- 中期 checkpoint 在 FID / LPIPS / art-FID 上最优
- 后面 style 可能略升，但总体质量变差

### 代表案例：`exp_G1_edge_rush`

数据：

- `epoch 60`
  - style `0.6656092909`
  - lpips `0.3803167425`
  - fid `282.9816017`
  - art_fid `399.1228862`
- `epoch 120`
  - style `0.6669374526`
  - lpips `0.3953184340`
  - fid `283.6427345`
  - art_fid `400.6075285`
- `epoch 150`
  - style `0.6660482574`
  - lpips `0.3908338525`
  - fid `284.6251857`
  - art_fid `400.8148226`

解释：

- 这是一条非常稳定的主线曲线
- 60 epoch 已经达到最优平衡
- 后面继续训练基本没有带来真正收益

### 代表案例：`micro05_id_anchor`

- `epoch 20` 是 FID / art-FID 最优
- `epoch 80` 只是 style 更高

解释：

- micro-run 更容易早出甜点
- 很适合快速筛选方向

## 3. 类型 B：style 持续上升、内容持续变坏型

定义：

- style 随训练上升
- LPIPS / FID / art-FID 同时变坏

### 代表案例：`style_oa_5_lr5e4_wc2_swd60_id30_e120_interval10`

- `epoch 30`
  - style `0.6981279703`
  - lpips `0.4249562226`
- `epoch 100`
  - style `0.7297230264`
  - lpips `0.5095066082`
- `epoch 120`
  - style `0.7264517831`
  - lpips `0.5217467501`

解释：

- 这类曲线说明参数配方有风格推力，但缺少足够的后期约束
- 训练目标更像“把 style 往上顶”，而不是维持均衡

### 代表案例：`abl_no_adagn`

- `epoch 40`
  - style `0.6855056301`
  - lpips `0.5233671738`
  - fid `304.9437813`
- `epoch 80`
  - style `0.6973906821`
  - lpips `0.5414757411`
  - fid `312.5094776`

解释：

- 去掉 AdaGN 后，模型会沿着“style 分数更高但画面更差”的方向漂移

### 代表案例：`abl_naive_skip`

- `epoch 40`
  - style `0.6984118814`
  - lpips `0.5939234347`
  - fid `326.3586001`
- `epoch 80`
  - style `0.7061084746`
  - lpips `0.6041445841`
  - fid `330.4055816`

解释：

- 未过滤 skip 非常容易产生这种曲线

## 4. 类型 C：稳定平台型

定义：

- 不同 checkpoint 几乎不变
- 说明模型早早进入某个固定解

### 代表案例：`abl_no_residual`

- `epoch 40`
  - style `0.6285932735`
  - lpips `0.2966944365`
  - fid `286.4353868`
- `epoch 80`
  - style `0.6285974741`
  - lpips `0.2965136992`
  - fid `286.4885003`

解释：

- 不是后期过拟合，而是结构表达上限本身就低

## 5. 类型 D：双 checkpoint 取舍型

定义：

- style 最佳和 FID 最佳来自不同轮次
- 但两者都可被辩护

### 代表案例：`G0-Base-Gain0.5`

- `epoch 40`
  - style `0.7024705430`
  - lpips `0.5263103249`
  - fid `313.5398288`
  - art_fid `486.8323859`
- `epoch 80`
  - style `0.7022560981`
  - lpips `0.5300168207`
  - fid `300.8889251`
  - art_fid `466.0451282`

解释：

- style 几乎不变
- 但后期 FID / art-FID 明显改善
- 这类实验适合按目标任务选 checkpoint，而不是机械选 style 最高轮

## 6. 类型 E：单点评估型

定义：

- 只有一个 `summary_history` 轮次
- 无法判断动态，只能视为定格证据

### 代表案例

- `exp_3_macro_strokes`
- `master_sweep_01_cap_64`

解释：

- 它们仍然有价值
- 但更像最终快照，不像训练曲线证据

## 7. 类型学的实际用途

这份文档后续可以直接用于：

1. 解释为什么同一实验不能默认最后一轮最好。
2. 判断某模块带来的问题是“训练后期漂移”还是“表达能力天花板”。
3. 选择哪些实验值得继续做更细的 epoch 级扫描。

## 8. 一个工作结论

当前最常见、最危险的失败类型不是“完全训练不动”，而是：

- style 继续涨
- 其它质量指标慢慢坏掉

也就是说，这个项目更像是在和“风格分数驱动的慢性漂移”作战。
